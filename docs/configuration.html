Configuration¶

Autonet uses the config-engine package as it’s application configuration provider. ConfigEngine allows Autonet to read configuration from multiple sources as required. For more information on how ConfigEngine works, please see the documentation. Autonet requires a minimal amount of configuration to get going. Autonet’s built in authentication database must be configured to store users and token data.

Additional configuration may be required for various backend and device drivers. See the documentation for those drivers for more details.
